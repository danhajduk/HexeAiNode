# AI-Node UI Redesign Tasks

## Scope

Redesign the AI-Node UI after the initial identity screen.

Keep the current initial setup screen that asks for:
- node name
- Core IP / Core endpoint

Redesign everything after that point.

Primary goals:
- separate setup UI from operational UI
- reduce dashboard clutter
- move advanced diagnostics out of the main operational view
- improve readability of capability and runtime information
- create a clearer state-driven UX

---

## Design Principles

- Setup is a guided flow, not a dashboard.
- Operational mode is a dashboard, not a setup wizard.
- Diagnostics are advanced tools and must not dominate the default view.
- Each piece of information should have one primary home.
- Raw payloads and verbose metadata should be collapsed or hidden behind diagnostics.
- The UI must remain functional for dark theme and desktop-first layouts.
- Existing initial node-name/Core-IP screen remains unchanged unless required for routing integration.

---

## Required Outcome

The AI-Node frontend must support three distinct UI layers:

1. Initial identity screen
   - existing screen
   - remains as-is

2. Setup UI
   - shown when the node is not fully operational
   - step-based flow
   - focused on onboarding, provider setup, declaration, governance, readiness

3. Operational UI
   - shown when the node is operational or degraded
   - dashboard-first layout
   - separate diagnostics section/page/tab

---

## State Rules

Use explicit UI mode resolution driven by lifecycle/runtime state.

Recommended mapping:

### Setup Mode
Show setup UI when state indicates unfinished onboarding or unfinished readiness, including cases such as:
- pending bootstrap
- pending registration
- pending approval
- pending trust activation
- capability_setup_pending
- governance pending
- setup incomplete
- provider not configured

### Operational Mode
Show operational UI when state indicates:
- operational
- degraded

### Manual Access
Operational UI must expose actions to:
- reopen setup/reconfigure flow
- open diagnostics

Setup UI may expose:
- restart setup
- exit to dashboard only if safe and state allows it
